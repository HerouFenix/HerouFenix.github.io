About Me

My name is Diogo Silva. I'm currently a student pursuing a Master's in "Engenharia Informática e de Computadores" (Software Engineering), specializing in Artificial Intelligence and Games now enroled on my final year at Instituto Superior Técnico (University of Lisbon).

I like to think of myself as someone who is proactive and in constant search of self-improvement both as a person and skill-wise. As such, I'm not afraid of getting any form of feedback, rather, I embrace all critiques and mistakes I make and use them in order to further improve through their respective analysis. Always more than willing and open to learning new technologies and methodologies, I aspire to contribute to a favorable and fulfilling work environment that benefits not only the workflow but also the team's synergy.

I've been pursuing a job in the Video Games' industry ever since I was a child. For as long as I can remember, the medium's always been a deep passion of mine and has accompanied me in life hand in hand. That passion soon grew to curiosity, making me want to learn and figure out how those expansive worlds, those gripping stories, that immersive gameplay were crafted and how they all came together, synergizing to form something so marvelous. Besides Video Games, I found a fun, cathartic activity in programming after a friend lent me a book with Python programming exercises. Merging those two passions together were the catalyst that led me down the path of Software Engineering, which I hoped to one day be the doorway into the job I had been dreaming of for so long.

Through my years at University, I've also cultivated an interest in Artificial Intelligence, Intelligent Systems and Machine Learning, alongside topics of Computer Graphics and 3D Programming, having worked on several projects encompassing a multitude of topics related to these areas, from Cooperative Agents, to Data Analysis ML models and full-blown Graphical Engines.

Other hobbies and interests of mine include most other forms of art such as music and illustration. I've been playing guitar and piano for a while now and absolutely love digital drawing. Besides Video Games, I also love Table Top RPGs such as D&D and Pathfinder. Now you know some stuff about me!

P2P Drive-Thru using Token Ring

On the second semester of my second year of University we had a course called "Distributed Computation" (aka Distributed Systems). On it we approached, studied and deeply followed all concepts in the book "Distributed Systems" by Andrew S. Tanenbaum and Marteen van Steen. As a first project we were told to create a "P2P Drive-Thru": A collection of several entities each running as it's own project and connected to each other in a Ring Overlay formation. All entities interacted with each other and, to manage the P2P communication/allow a process to take control and request some work/access critical resources (i.e resources that can only be used by one process at a time) we had to construct a "Token-Ring" - Using a token that continously circulates through the ring, a process gets the token, checks if the tokens empty, if so, checks if it has something to request or a message to send, puts it in the token (if it does) and passes it on to the next entity in the ring ; if the token isn't empty, the entity checks if the request on it is for itself, if so it gets the request, processes it and passes the token on, else, it simply fordwards the token. Simple

TEAM: Vasco Ramos TECH: Python Program, Distributed Systems, P2P Communication, Token Ring

Word Counter with Map Reduce and Fault Taulerance

Our second project for the "Distributed Computation" subject we were taught about concepts related to Fault Tolerance and Consistency and Replication. To hone our newly gained knowledge in these areas we were told to create a...word counter. A simple program that would get a text (or a book) and count how many instences of each word there were. "Easy", we thought, I mean, a word counter is something you make when you first learn about programming. Here's the thing though: Counting the words in huge texts (like the Bible) isn't too efficient if you're just using a single process. So the meat of the project was to manage several processes counting different chunks of the text and have them synergize with each other using a single Coordinator, who would break the text into several partitions and distribute the work accordingly, and several Workers who would actually count the words in the chunk they were given by the coordinator, all using a Map Reduce Algorithm. Communication was handled using Asyncio. The pain came from the fact that each process should be able to crash/fail (either the Coordinator or the Workers) without that affecting the final result, which we achieved through a backup coordinator and Asyncio's capabilities to detect connection breaks.

TEAM: Vasco Ramos TECH: Python, Asyncio Program, Distributed Systems, P2P Communication, Map-Reduce

dimX

We had a subject called "Compilers" during our 2nd year of Uni which was all about the development and conceptualization of Programming Languages, the concepts behind formal languages and how a compiler does it's work. Very interesting stuff but at the time we found the subject absolutely dreadful. Nevertheless we worked as hard as we could to fulfill our final project which envolved creating a language using ANTLR4 to define it (i.e a Lexer) and Java (through the use of Listeners and Visitors to walk the Syntax Tree) to add logic (i.e a Parser). Our language was all about the ability the define Physical Quantities and Dimensions in order to allow for their computation (for example, a person would be able to compute the value of 10 meters divided by 2 seconds, which would output 5 m/s ; however summing 10 meters and 2 seconds would output an error). Our language would get converted to Java code, which would then be automatically compiled with the normal Java Compiler.
